# Diagrams

Visual representations make complex relationships immediately understandable — for you, for collaborators, and for Claude. This folder stores standalone diagram files. Diagrams that belong to a single document (e.g., the ERD in `DATA_MODEL.md`) can live inline in that document instead.

---

## Using Mermaid

All diagrams in this framework use [Mermaid](https://mermaid.js.org/), a plain-text diagramming syntax that:
- Renders natively in GitHub Markdown
- Can be generated by Claude
- Can be exported to images for presentations and documents
- Lives in version control alongside your docs

---

## How to Ask Claude to Generate a Diagram

Be specific about what you want to visualize. Examples:

**For a data model ERD:**
> "Based on the entities in my DATA_MODEL.md, generate a Mermaid ERD showing the relationships."

**For a user journey flowchart:**
> "Create a Mermaid flowchart for the sign-up journey described in USER_JOURNEYS.md."

**For a system architecture diagram:**
> "Draw a Mermaid architecture diagram showing the components in ARCHITECTURE.md."

**For a sequence diagram:**
> "Create a Mermaid sequence diagram showing what happens when a user submits a form — including the frontend, backend, and database steps."

---

## Mermaid Diagram Types and When to Use Them

| Type | Syntax opener | Best for |
|------|--------------|----------|
| Flowchart | `flowchart TD` | User journeys, decision logic, process flows |
| Entity Relationship | `erDiagram` | Data models |
| Sequence | `sequenceDiagram` | System interactions, API calls, event flows |
| Architecture | `architecture-beta` | High-level system components |
| Gantt | `gantt` | Project timelines and milestones |
| Pie chart | `pie` | Simple proportional data |

---

## Exporting Diagrams

To export a Mermaid diagram to an image (for slides, Google Docs, etc.):
- Use the [Mermaid Live Editor](https://mermaid.live) — paste your diagram code and download as PNG or SVG
- In VS Code, use the Mermaid Preview extension
- Claude can also describe how to export using the CLI tool `mmdc` if you need automation
