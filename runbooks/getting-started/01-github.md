# Runbook: Create a GitHub Account

GitHub is where your project's code and documents live. It's version control, backup, collaboration, and release management — all in one place. This runbook walks you through the minimum setup needed to use the Hacky Hours workflow.

---

## What You'll Need

- An email address
- A web browser

---

## Steps

### 1. Create an account

Go to [github.com](https://github.com) and click **Sign up**. Choose a username that you'd be comfortable sharing — it will appear on your public projects.

### 2. Verify your email

GitHub will send a verification email. Click the link to confirm your account.

### 3. Choose a plan

The **Free** plan is sufficient for everything in this framework. You can upgrade later if you need private repositories for a team.

### 4. Enable two-factor authentication (required)

GitHub requires 2FA. Go to **Settings → Password and authentication** and set up either an authenticator app (recommended) or SMS. Save your recovery codes somewhere safe — losing them can lock you out of your account.

### 5. Create your first repository

A repository (repo) is a folder for your project. To create one:
1. Click the **+** icon in the top right → **New repository**
2. Name it after your project (lowercase, hyphens instead of spaces — e.g., `my-project`)
3. Set visibility to **Private** unless you intend this to be public from the start
4. Check **Add a README file**
5. Click **Create repository**
